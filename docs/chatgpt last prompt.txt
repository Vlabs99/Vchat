I am building a full Android realtime messaging application called VChat using Java + Firebase Firestore + Firebase Authentication + Firebase Storage in Android Studio.

The app is inspired by WhatsApp / Instagram DMs / Messenger.

Current architecture already includes:

* direct chats
* group chats
* friend system
* friend requests
* remove/block/unblock
* reply system
* forward system
* typing indicators
* pinned messages
* realtime Firestore listeners
* pagination
* delete for me
* group admin messaging
* restriction banners
* modular managers/helpers

Major stabilization work already completed:

1. Chat delete/reopen lifecycle fixed using:
   chatDeletedAt.<uid>
   Old hidden messages stay hidden while new activity appears correctly.

2. Relationship system heavily stabilized:

   * centralized applyRestriction(state)
   * normalized relationship states
   * precedence system:
     FRIEND > PENDING > REMOVED > UNKNOWN
   * listener safety guards
   * duplicate listener prevention
   * thread-safe UI updates

3. Mirrored relationship write issue fixed:
   Runtime tracing discovered:
   PERMISSION_DENIED during mirrored writes.
   Firestore rules updated to allow mirrored peer relationship writes safely.

4. Group resurrection fixed:
   Deleted/hidden groups now reappear automatically when new messages arrive while preserving hidden old history.

5. Listener stabilization completed:
   relationship/message/presence/pinned/group listeners protected from duplicate registrations and memory leaks.

Current relationship philosophy:

* If X removes Y:
  only X can resend request.
* If Y rejects X's retry:
  only Y can send the next request.
